Recordings, transcripts & whiteboard — new
Cloud recording
Built-in API
  • Cloud recording is a single API flag — set start_cloud_recording: true when creating the room token and recording starts automatically. Outputs 1080p MP4 retrievable via REST API or dashboard.
Requires setup
  • Cloud recording requires Jibri, a separate server component you must install and maintain yourself. Without it, only local (browser-download) recording is available. JaaS paid tier includes cloud recording without Jibri.
Supported
  • Cloud recording is available via the Video SDK with a single API call to startCloudRecording(). Files are retrievable via REST API as MP4. Costs extra on top of per-minute usage.
Via Graph API
  • Recordings are stored in OneDrive/SharePoint and accessible via Microsoft Graph API after the meeting. Setup requires Azure AD app registration and specific permissions.
Transcripts
First-class API
  • Real-time transcription powered by Deepgram is built in. Transcripts are stored automatically and retrievable via webhook or REST. You can also run batch transcription on any recording via the Batch Processor API — passing a recording ID returns a transcript file.
Requires config
  • Jigasi (a separate component) is needed for live transcription on self-hosted Jitsi. JaaS includes it. The Iframe API has a startRecording command with a transcription flag, but getting usable transcript files out requires additional configuration.
Supported
  • Transcripts are available via the cloud_recording_transcript_option JWT field. A webhook fires when the transcript is complete. Community reports of the audio transcript toggle being missing in some account configurations — a known pain point for developers.
Post-meeting only
  • Transcripts are accessible via Graph API after the meeting ends as a VTT file. Live transcript access during an active meeting is not currently available via API — only visible inside the Teams interface itself.
Screen & whiteboard recording
Included in cloud recording
  • Cloud recording captures whatever is on screen — video, shared screen, and whiteboard — in a single composited MP4. You can also record raw individual tracks separately for maximum flexibility. No extra steps needed to capture screen shares.
Manual or Jibri
  • Jibri captures everything visible in the conference including screen shares and whiteboard, but only works on self-hosted Jitsi. Community-built browser recording hacks exist but are not production-grade. The Iframe API exposes a whiteboard toggle but capturing its content as part of a recording requires Jibri.
Supported
  • Cloud recording can capture shared_screen_with_speaker_view as a separate MP4 file. Zoom also has a dedicated Whiteboard API, though whiteboard content is stored separately and not automatically merged into the meeting recording.
Partial
  • Teams records screen shares as part of the meeting recording. However, Whiteboard content created in Teams is stored in a separate OneDrive file and is not captured in the meeting recording video — you'd need to retrieve it separately via the Whiteboard API.
AI-ready output
Best pipeline
  • The Batch Processor API accepts a recording ID and returns a transcript in one call — purpose-built for feeding into AI. Webhooks notify you the moment a transcript is ready so you can trigger your summary pipeline automatically.
DIY pipeline
  • No native AI pipeline. You collect the recording file and transcript separately, then build your own pipeline to feed them into an AI service. Workable but more assembly required.
Webhook-driven
  • Webhooks fire when recordings and transcripts are complete, giving you a download URL to pull into your own AI pipeline. Functional but adds integration complexity compared to Daily.co's Batch Processor.
Complex pipeline
  • Requires Graph API subscriptions, webhook setup, Azure AD permissions, and separate handling for recording and transcript files. Significant engineering effort before you can feed data into an AI summarizer.
Overall fit
Best for
Building a tutoring platform that needs reliability, branding control, clean student experience, and the simplest possible path to AI-powered session summaries.
Getting started fast with zero cost while the platform is in development — then graduate to JaaS or swap to Daily.co when recordings and transcripts become critical.
Tutors and students who already have Zoom accounts and the app installed, and where the extra per-minute cost is acceptable.
Organizations already deep in Microsoft 365 where Teams is the standard tool and the complex API setup is already a sunk cost.
